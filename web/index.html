The Core Directives

VoidChat isn't a "platform." It’s an ephemeral instance. If you aren't actively in the tab, the chat doesn't exist.

Operational Flow

Establishing a connection is a two-step handshake:

  • Handshake: Select a handle. If you're hosting, you decide if joiners need explicit approval.
  • Broadcast: Generate your unique room link. Share it via your preferred secure channel.

Host-only settings like Kick and Deny are handled locally in-memory. Once the host leaves, the room is effectively decommissioned.

Static & Silent
This page is inert. No background networking, no tracking, no cookies. Just code.
Explicit Opt-In
Nothing connects until you click the button. No auto-joining, ever.
Volatile Memory
Closing the tab wipes the WebRTC data channels. No history is written to disk.

Specs & Constraints

  • Direct P2P: Messages travel browser-to-browser via WebRTC.
  • Introductions Only: A minimal signaling relay is used for the initial handshake, then gets out of the way.
  • No Identity: No accounts, no persistent IDs, no "who's online" directory.
  • The Vibe Check: If a feature requires a database, we don't build it.

Defensive Measures

Privacy isn't just about encryption; it's about friction.

  • Host Kick: Hosts can terminate a peer's connection and apply a local denylist cooldown.
  • Room Purge: The host can end the session for all participants instantly.
  • Rate Limits: Basic spam protection to prevent the room from melting.

Safe Operating Procedures

Remember, safety is rule #1: Even if the code is ephemeral, a screenshot is forever.

  • Assume peers can capture the screen.
  • Only share links with trusted entities.
  • If you don't recognize the host ID, don't authorize the connection.

FAQ

Does this page start a process?
Negative. This is just documentation until you move to the chat sub-route.

Is there a persistence layer?
Zero. No databases, no logs, no recovery. Lost messages stay lost.

Can I join a public lobby?
There is no lobby. VoidChat is link-only. No discovery.
